This is a Free base template for a Django & React application, it comes with a few simple applications and a well organized project structure.

The Project Includes


Frontend JavaScript build pipeline, which uses Webpack to bundle the JavaScript file ./asset/JavaScript/site.js The Webpack is configured to translate; .js | .jsx and bundle them into a browser readable format in ./static/JavaScript/site-bundle.js

RestFul API, which uses DRF, this helps you get your API's up and running fast. An example of creating an API from a excel file ./applications/api/views.py

MarkDown/HTML .md Content Management System, This app serves .md files to your Django Templates, through the ./applications/content/views.py this is used to create articles and content for your project. This is a very simplistic method for CMS but that why it's good, because you have full control over the content and delivery. Their are many floors, such as having to re-commit your application in production but we suggest concentrating on the quality of your SEO content rather than quantity.

Django: SEO & Sitemap which uses Django to create a /sitemap.xml page, which you can register with Search Engines like GOOG, to add urls to your http://127.0.0.1:8000/sitemap.xml Vist ./applications/website/SiteMap.py Meta Tags can be overridden in the templates, for an example ./templates/applications/content/content_django.html The Default html meta tags should be written in the 'django blocks' on the ./templates/base.html page this serves as the base html page for the project.

Website Django Application This app is for any generic webpages; terms, privacy, about etc. website templates are basically for static pages / backend rendered pages.

The idea with this is it allows you to build a your application rapidly as some of the vital infrastructure is already layed in a well structured way following good django & web practices. You can easily integrate react into any of your django templates give you the option to use modern JavaScript or not.
